# 检测效果优化指南

## 🔍 如果检测不到物体

### 1. 检查训练数据
```bash
# 查看训练了多少样本
python -c "import pickle; m=pickle.load(open('models/video_trained_model.pkl','rb')); print(f'样本数: {m[\"total_samples\"]}')"
```

**建议：**
- 每个类别至少 50 个样本
- 如果样本太少，重新录制更多视频

### 2. 改善检测环境

**光照：**
- ✅ 充足、均匀的光照
- ❌ 避免强烈阴影
- ❌ 避免逆光

**背景：**
- ✅ 简洁的背景
- ❌ 避免杂乱背景
- ❌ 避免与物体颜色相近的背景

**物体：**
- ✅ 清晰可见
- ✅ 完整展示
- ❌ 避免遮挡
- ❌ 避免模糊

### 3. 调整摄像头

**距离：**
- 物体占画面的 20-50%
- 不要太近（失焦）
- 不要太远（特征不清）

**角度：**
- 与训练时的角度相似
- 避免极端角度

### 4. 重新训练

如果检测效果不好，收集更多数据：

```bash
# 1. 录制更多视频（不同角度、光照）
./run_video_annotator.sh

# 2. 重新训练
source venv/bin/activate
python src/train_from_video.py

# 3. 测试检测
./run_simple_detector.sh
```

## 🎯 测试检测效果

### 方法 1：使用训练视频测试
```bash
# 播放训练时录制的视频
# 看看能否检测到
```

### 方法 2：逐步测试
1. 先测试训练时的物体（应该能检测到）
2. 再测试相似的物体
3. 最后测试不同角度的物体

### 方法 3：查看匹配点数
- 检测框下方显示 "Matches: XX"
- 匹配点越多，检测越可靠
- 如果匹配点 < 10，说明特征不够

## 🎬 完整工作流程

```bash
# 1. 标注（录制 2-3 个视频，每个 20-30 秒）
./run_video_annotator.sh

# 2. 训练
source venv/bin/activate
python src/train_from_video.py

# 3. 检测
./run_simple_detector.sh

# 4. 如果效果不好，回到步骤 1 录制更多视频
```

## ✅ 检测成功的标志

- 绿色框准确标注物体
- 置信度 > 60%
- 匹配点数 > 20
- 框的位置稳定（不抖动）

## ❌ 检测失败的原因

1. **训练样本太少** - 录制更多视频
2. **环境差异大** - 保持环境一致
3. **物体特征不明显** - 选择纹理丰富的物体
4. **遮挡严重** - 确保物体完整可见
5. **移动太快** - 放慢移动速度
