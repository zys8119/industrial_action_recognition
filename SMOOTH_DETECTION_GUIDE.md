# 解决检测框抖动问题

## 🎯 问题原因

检测框抖动是因为：
1. 每帧检测的特征点略有不同
2. 匹配结果有微小偏差
3. 边界框位置在小范围内波动

## ✅ 解决方案

已添加**时间平滑算法**：
- 记录最近 5 帧的检测位置
- 计算平均坐标
- 输出平滑后的边界框

## 🚀 使用方法

### 默认平滑（推荐）
```bash
./run_detector.sh
```

### 调整平滑强度

```bash
# 更强的平滑（更稳定，但响应慢）
./run_detector.sh --smooth 10

# 更弱的平滑（响应快，但可能抖动）
./run_detector.sh --smooth 3

# 无平滑（原始检测）
./run_detector.sh --smooth 1
```

## ⌨️ 快捷键

- **Q** - 退出
- **S** - 截图
- **R** - 重置平滑缓存（如果检测卡住）

## 💡 其他减少抖动的方法

### 1. 提高标注质量
- 标注更多样本（10+ 个）
- 包含不同角度和位置
- 确保特征点丰富

### 2. 改善环境
- 稳定光照
- 固定摄像头
- 减少背景干扰

### 3. 调整检测参数
```bash
# 提高匹配要求（更稳定但可能漏检）
./run_detector.sh --min-matches 15 --confidence 0.7

# 降低匹配要求（检出率高但可能抖动）
./run_detector.sh --min-matches 5 --confidence 0.4
```

### 4. 使用跟踪工具
如果需要更稳定的跟踪，使用专门的跟踪工具：
```bash
./run_tracking.sh
```

## 🔧 技术细节

### 平滑算法
```python
# 记录最近 N 帧的检测位置
history = [bbox1, bbox2, bbox3, bbox4, bbox5]

# 计算平均坐标
avg_x = mean([b.x for b in history])
avg_y = mean([b.y for b in history])
avg_w = mean([b.w for b in history])
avg_h = mean([b.h for b in history])

# 输出平滑后的框
smoothed_bbox = (avg_x, avg_y, avg_w, avg_h)
```

### 优点
- ✅ 显著减少抖动
- ✅ 保持检测准确性
- ✅ 计算开销小

### 缺点
- ⚠️ 响应延迟（约 N/2 帧）
- ⚠️ 快速移动时有拖尾
