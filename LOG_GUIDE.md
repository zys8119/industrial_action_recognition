# 日志功能说明

## 📝 日志记录

摄像头 Demo 现在会自动记录所有检测结果到日志文件。

### 日志位置
```
logs/recognition.log
```

### 日志内容
- 会话开始/结束时间
- 每次动作检测的详细信息：
  - 动作类别
  - 置信度
  - 位置坐标 (x1,y1)-(x2,y2)
  - 边界框尺寸
- 帧数统计
- 系统事件（重置、截图等）

### 日志格式示例
```
[2026-03-10 11:08:30.123] === 会话开始 2026-03-10 11:08:30 ===
[2026-03-10 11:08:30.456] ✅ 摄像头已打开
[2026-03-10 11:08:31.789] 检测到动作: welding | 置信度: 85% | 位置: (120,180)-(320,380) | 尺寸: 200x200
[2026-03-10 11:08:32.012] 检测到动作: assembly | 置信度: 92% | 位置: (450,200)-(650,450) | 尺寸: 200x250
[2026-03-10 11:08:35.345] 🔄 缓冲区已重置
[2026-03-10 11:08:40.678] 📸 截图已保存: screenshots/screenshot_20260310_110840.jpg
[2026-03-10 11:10:00.901] === 会话结束 2026-03-10 11:10:00 ===
[2026-03-10 11:10:00.902] 总帧数: 1800
```

## 🔍 查看日志

### 1. 查看最近日志
```bash
cd ~/Desktop/industrial_action_recognition
source venv/bin/activate

# 查看最后 20 行
python src/view_log.py

# 查看最后 50 行
python src/view_log.py --tail 50
```

### 2. 实时监控日志
```bash
# 实时显示新增日志（类似 tail -f）
python src/view_log.py --watch
```

### 3. 分析日志统计
```bash
# 生成统计报告
python src/view_log.py --analyze
```

## 🎨 画面显示

### 检测框颜色
- **绿色** - 高置信度 (>80%)
- **黄色** - 中等置信度 (60-80%)
- **橙色** - 低置信度 (<60%)

### 显示信息
- 边界框：标注动作发生的区域
- 标签：动作类别 + 置信度百分比
- 中心点：检测框的中心坐标
- 坐标文字：实时显示位置

### 顶部信息栏
- 整体动作识别结果
- 当前检测数量
- 帧计数
- 缓冲区状态
- FPS

## ⌨️ 快捷键

- **q** - 退出程序
- **r** - 重置缓冲区和检测框
- **s** - 截图保存（保存到 screenshots/ 目录）

## 📸 截图功能

按 `s` 键保存当前画面（包含所有标注）：
```
screenshots/
├── screenshot_20260310_110840.jpg
├── screenshot_20260310_110845.jpg
└── ...
```

## 🚀 运行示例

### 启动识别（带日志）
```bash
cd ~/Desktop/industrial_action_recognition
./run_camera.sh
```

### 启动识别（禁用日志）
```bash
source venv/bin/activate
python src/camera_demo.py --no-log
```

### 指定日志文件
```bash
python src/camera_demo.py --log my_logs/test.log
```

### 同时查看日志
```bash
# 终端1：运行识别
./run_camera.sh

# 终端2：实时监控日志
cd ~/Desktop/industrial_action_recognition
source venv/bin/activate
python src/view_log.py --watch
```

## 💡 使用建议

1. **长时间运行** - 日志会持续追加，定期清理旧日志
2. **性能分析** - 通过日志分析检测频率和准确率
3. **问题排查** - 日志记录所有事件，便于调试
4. **数据收集** - 日志可用于后续的数据分析和模型优化
